# Overdrive Skill Summary

Generated from the current installed skill roots. This file is for humans; agents should use `skill-router` and its catalog for routing.

## Will Claude Code, Codex, Gemini, Antigravity, and Cursor use these automatically?

Yes, after install and restart/reload. Claude Code, Codex, Gemini CLI, Antigravity, and Cursor discover skills by reading `SKILL.md` files from their configured skill roots. The installer copies valid skill folders into the selected global or local roots and `verify.sh` checks frontmatter, broken symlinks, missing skills, forbidden Composio/Google Drive automation, and router smoke prompts.

They do not need to manually link each skill. The only manual step is restarting/reloading the app or opening a fresh session so the runtime re-indexes the skill list. Overdrive v2 keeps the current 137-skill manifest, documents native context-window behavior by runtime, adds the ovd-plan project-management layer, and can attempt safe optional setup for a few external helpers when selected, including Graphify, Design Extract browser support, Claude Video helpers, and `yt-dlp` for media downloads. Account-based tools and keys still remain user-configured. See `docs/skill-readiness.md` for the plug-and-play audit.

This kit is upstream-first, pinned by default, and target-aware: normal installs pull third-party skills from verified GitHub refs or pinned official npm installers, then install only into selected/detected targets using a non-destructive conflict policy. Public releases do not ship vendored third-party skill snapshots.

ovd-workflow is the non-skill project-state layer added in v0.4 and expanded through v0.8. It creates a local, gitignored `.overdrive/` folder for active work, decisions, preferences, research, file hashes, route traces, reports, handoffs, and a small knowledge vault. Skills teach the agent how to do work; ovd-workflow helps it remember what is happening in a specific project without bloating every prompt.

## How The Skills Work Together

- Start with `skill-router` as the default lightweight preflight for non-trivial requests; it should pick the minimum relevant skill sequence rather than loading everything. Complex tasks can use more skills when phased clearly.
- Use ovd-workflow as local project memory when `.overdrive/` exists: read only the state, preferences, active work, or relevant knowledge-vault index entry that helps the task, and use `status`, `doctor`, `resync`, `knowledge`, or `checkpoint` when continuation/health/handoff/reference-doc indexing matters.
- Global/project instructions add always-on behavior across Claude Code, Codex, Gemini CLI, Antigravity, and local project agents: think before coding, keep solutions simple, make surgical changes, keep context lean, verify against explicit goals, consult `skill-router` before non-trivial work, and use Context7 for current docs when it fits the task.
- Clarification and planning: `clarify-and-plan` handles ambiguity, assumptions, options, and phase boundaries. `planning-first` wraps complex non-Claude coding work in Explore -> Plan -> Implement -> Verify. Claude Code users can prefer native `/model opusplan` and `/ultraplan`, but should not run duplicate planning passes.
- Prompt engineering: `prompt-master` is for writing or improving prompts themselves. It is not a replacement for `clarify-and-plan`, which clarifies the actual work the agent is about to perform.
- Security and launch: Claude Code should use native `/security-review`; other agents use the portable `security-review` skill. `pre-launch-checklist` covers product/business launch readiness and pairs with focused security, SEO, marketing, or browser checks.
- Research and onboarding: `last30days` handles recent community/social research; `reddit-research` handles explicit Reddit/subreddit public signal; `app-onboarding-questionnaire` handles questionnaire-style app onboarding flows before implementation.
- Product design layers: `layers-intro` plus the narrow `layers-*` skill helps agents reason about observed behaviour, domain language, user needs, product strategy, conceptual models, interaction flows, and surface decisions before jumping into implementation.
- Frontend/design: Taste gives visual direction, Emil handles broad motion/feel, Emil Animation Polish handles CSS transition/easing details, Fluid Animations handles spring/gesture/direct-manipulation behavior, Modern Web Guidance checks current platform APIs, Playwright verifies in browser, and Impeccable does final polish with user feedback before broad hierarchy/font changes.
- Glass UI: `liquid-glass-web` chooses between universal frosted glass, Chromium SVG displacement, and optional WebGL refraction based on target browsers, performance budget, and backdrop constraints.
- Text layout engineering: `pretext` is for text measurement/layout performance, such as virtualized variable-height text rows, shrinkwrapped chat bubbles, multiline layout without DOM reflow, and Canvas/SVG/WebGL text. It is not a visual-polish substitute.
- Design extraction: `design-extract` ingests public website design language when optional tooling is available, then feeds Taste/Impeccable/Emil implementation or polish rather than replacing those skills.
- Premium 3D website workflows: Jack Roberts inspired local skills handle brand/competitor intelligence, scroll-stop image/video prompts, scroll-driven video/frame-sequence implementation, SEO/launch audits, and full workflow orchestration. They treat Firecrawl, GitHub, Vercel, image/video tools, and browser connectors as optional user-configured tools.
- Marketing/content: Corey Haines skills handle the specific channel or growth problem; Stop Slop is the final voice pass for public-facing prose.
- Human voice cleanup: `humanizer` edits existing text to preserve meaning and facts while making it sound more natural; it should not be used for deceptive authorship or fake lived experience.
- Objective review and React code health: `what-should-i-consider` pressure-tests plans before implementation; `react-doctor` handles React-specific diagnostics and cleanup.
- Knowledge work: `convert-to-markdown` converts local PDFs, Office files, spreadsheets, HTML exports, and data files to Markdown before reading; ovd-workflow `knowledge/` keeps local reference docs indexed; retained Obsidian-adjacent skills handle JSON Canvas and Defuddle extraction.
- Code/corpus intelligence: `graphify` maps and queries codebase or mixed-corpus relationships when the optional `graphifyy` tool is available; the installer attempts safe setup, and agents fall back to `rg`, file reads, tests, and ovd-workflow state when it is unavailable. Do not route both Graphify and ovd-workflow knowledge vault unless both relationship mapping and project-memory/reference-doc context are actually needed.
- Media utilities: `claude-video` watches or analyzes videos and screen recordings when optional dependencies are available; the installer attempts safe `ffmpeg`/`yt-dlp` setup. `media-download` wraps yt-dlp for user-requested MP3 and high-quality MP4 downloads, defaulting to local folders and permission-aware behavior.
- Anthropic examples: Brand Guidelines is for explicitly Anthropic-branded artifacts, Doc Co-Authoring is for substantial docs/specs/proposals, MCP Builder is for MCP server design/implementation, and Slack GIF Creator is for Slack-ready animated GIFs or emoji.
- External actions: Composio/connect-style skills are approval-gated. The agent must ask before sending, posting, creating, deleting, authenticating, or spending credits.
- MCPs/connectors: Context7 docs guidance transfers. Other repository, database, browser, or app connectors are intentionally personal/project-specific. Credentials, tokens, OAuth sessions, service-role keys, and MCP configs do not transfer.
- Image/video/media: Banana is preferred for image generation where configured; Remotion handles React/programmatic video; `media-download` handles user-requested yt-dlp downloads; Playwright validates browser-visible results.

## Onboarding Checklist

1. Unzip or clone the kit.
2. Run `./install.sh --dry-run`.
3. Run `./install.sh`.
4. Run `./verify.sh`.
5. Restart Claude Code, Codex, Gemini CLI, and Antigravity.
6. Sign in or configure only the external tools you actually intend to use.

## Updating

- Run `./update.sh` from a cloned repo to pull the latest Overdrive setup, refresh Overdrive-managed skills from verified pinned sources, and verify the install.
- Run `./update.sh --all-skills` to refresh every matching skill from verified pinned sources with timestamped backups for replaced folders.
- Add `--allow-upstream-drift` only when you intentionally want tracking branches and latest package installers.
- Run `npx -y github:radustefandumitru/Overdrive update-skills` when using the GitHub `npx` path; add `--all-skills` for the backup-and-replace refresh.
- Run `./uninstall.sh --dry-run` to preview removing only Overdrive-marked skill folders and managed instruction blocks.

## Manual Dependencies

- Banana Claude may need Google/Gemini API setup.
- Obsidian CLI workflows need Obsidian open and the `obsidian` CLI installed/configured.
- Defuddle needs `npm install -g defuddle`.
- Playwright workflows may need Playwright installed and browsers available.
- Composio/connect and app connectors need account auth and explicit user approval before side effects.
